# UI Styling (shadcn/ui + Tailwind CSS)

## Stack
- **shadcn/ui**: Pre-built accessible components via Radix UI primitives (copy-paste model)
- **Tailwind CSS**: Utility-first CSS framework with build-time processing
- **Canvas**: Museum-quality visual compositions with philosophy-driven design

## When to Use
- Building React apps needing accessible components (forms, dialogs, tables)
- Implementing responsive mobile-first layouts
- Establishing consistent design systems
- Rapid prototyping with immediate visual feedback

## Setup
```bash
npx shadcn@latest init
```
Configures both shadcn/ui and Tailwind. Add components individually as needed.

## Component Categories
- **Forms**: Input, Select, Checkbox, Radio, Switch, Slider, DatePicker
- **Overlays**: Dialog, Sheet, Popover, Tooltip, DropdownMenu
- **Data Display**: Table, Card, Badge, Avatar, Carousel
- **Navigation**: Tabs, Breadcrumb, NavigationMenu, Pagination
- **Feedback**: Toast, Alert, Progress, Skeleton

## Key Patterns
- Keyboard navigation + ARIA support on all interactive elements
- Theme customization via CSS variables
- Dark mode with `class` strategy
- Responsive utilities: sm/md/lg/xl breakpoints
- Consistent spacing scale via Tailwind config

## Design Philosophy
Expert craftsmanship where every detail matters. UI development as a refined discipline balancing accessibility, performance, and visual excellence.
